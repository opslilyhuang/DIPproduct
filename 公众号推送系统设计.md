# Obsidian + AI 公众号推送系统设计

## 目标
基于文章《Agent生产力系列：Obsidian+Claudian打造公众号写作系统》的思路，构建一个自动化公众号内容创作和推送系统。

## 核心组件

### 1. 内容管理 (Obsidian)
- **Vault结构**：
  - `公众号/草稿/` - 写作中的文章
  - `公众号/已发布/` - 已推送的文章
  - `公众号/素材/` - 图片、引用等素材
  - `公众号/模板/` - 文章模板
- **文章格式**：Markdown + YAML frontmatter
  ```yaml
  ---
  title: 文章标题
  description: 文章描述
  tags: [标签1, 标签2]
  cover: 封面图URL
  publish_date: 发布日期
  status: draft/review/published
  ---
  ```

### 2. AI辅助写作 (Claudian/其他AI工具)
- **内容优化**：语法检查、风格调整、SEO优化
- **标题生成**：多个标题变体选择
- **摘要生成**：文章摘要
- **标签建议**：自动分类标签

### 3. 格式转换
- **Markdown → 公众号HTML**：
  - 处理公众号特有格式（居中、引用、分割线等）
  - 图片上传到图床（如腾讯云COS、七牛云）
  - 代码块转为图片或保持文本
- **预览生成**：本地HTML预览

### 4. 公众号集成
- **API对接**：微信公众平台API
- **草稿箱**：上传到公众号草稿箱
- **发布流程**：预览 → 审核 → 发布

## 工作流设计

### 阶段1：内容创作
1. 在Obsidian中新建文章 (`公众号/草稿/2026-03-11-文章标题.md`)
2. 编写Markdown内容，插入本地图片
3. 使用AI工具优化内容
4. 添加frontmatter元数据

### 阶段2：格式转换
1. 扫描文章中的本地图片
2. 上传图片到图床，替换为URL
3. 转换Markdown为公众号HTML格式
4. 生成本地预览

### 阶段3：预览与审核
1. 在浏览器中打开预览
2. 人工审核内容
3. 调整格式（如有需要）

### 阶段4：推送发布
1. 通过公众号API上传到草稿箱
2. 设置封面、摘要、标签
3. 发布到公众号（或定时发布）

## 技术实现方案

### 依赖技能/工具
1. **Obsidian技能**：已安装，管理vault和笔记（需要obsidian-cli）
2. **微信公众号技能组**（skillhub搜索发现）：
   - `wechat-article-writer`：公众号文章创作（选题到成稿+配图）
   - `wechat-article-typeset`：公众号文章排版（生成HTML和复制页链接）
   - `mp-draft-push`：公众号自动发布（推送到草稿箱）
   - `wechat-articles`：公众号文章搜索和读取
   - `fastfish-lite`：公众号快速排版（格式整理、敏感词检测、热点）
3. **AI辅助技能**：替代Claudian
   - 现有AI模型（DeepSeek等）进行内容优化
   - 或安装专门的AI写作技能
4. **图片处理技能**：需要图床上传功能（可集成到工作流中）
5. **Markdown转换工具**：可能需要自定义脚本或使用现有排版技能

### 开发任务

3. **自动化集成**：
   - 创建协调脚本，连接Obsidian和微信公众号技能
   - 实现文件监听和自动触发
   - 添加错误处理和状态监控

## 风险评估

### 技术风险
1. **公众号API限制**：每日调用次数限制，内容审核规则
2. **图片上传稳定性**：图床服务可用性
3. **格式转换准确性**：复杂Markdown元素的转换

### 内容风险
1. **AI生成内容合规性**：需要人工审核
2. **版权问题**：图片和引用来源
3. **发布时间**：避免敏感时段
